一、Hexo 项目基础配置
URL 与根目录配置：
当网站部署在 https://mizore-ni.github.io/project 时，_config.yml 应如下设置：
yaml
url: https://mizore-ni.github.io/project
root: /project/
permalink: :year/:month/:day/:title/
部署在根目录 https://mizore-ni.github.io 时：
yaml
url: https://mizore-ni.github.io
root: /
permalink: :year/:month/:day/:title/
本地开发时（假设本地服务器地址为 http://localhost:4000）：
yaml
url: http://localhost:4000
root: /
permalink: :year/:month/:day/:title/
Front - matter 示例：
yaml

title: My Blog Post
date: 2023-01-01
tags: [blog, example]
layout: post

二、资源路径相关
本地 CSS 文件查找：
对于本地服务器 localhost:4000 请求的 /project/css/style.css，通常在 Hexo 项目的 themes/[主题名称]/css 目录下查找 style.css 文件。
线上 CSS 文件查找：
若网站部署后，可能在 public/project/css 目录下找到 style.css 文件（假设基于 Hexo 生成的 public 文件夹部署）。
其他相关路径示例：
如 W:\my_blog.deploy_git\css 和 W:\my_blog\public\css 下的 style.css 文件，.deploy_git 可能与项目部署相关，public 是 Hexo 生成静态文件的常用目录。
三、网页显示异常排查
字体加载问题：
在 style.css 中，字体文件路径错误时：
css
@font-face {
font-family: ‘YourFontFamily’;
src: url(‘path/to/your/font.ttf’);
}
应确保 src 路径正确指向字体文件。
字体格式兼容性问题，应提供多种格式：
css
@font-face {
font-family: ‘YourFontFamily’;
src: url(‘path/to/your/font.woff2’) format(‘woff2’),
url(‘path/to/your/font.woff’) format(‘woff’),
url(‘path/to/your/font.ttf’) format(‘ttf’);
}
CSS 样式问题：
文字颜色与背景颜色相同导致不可见，例如：
css
body {
color: #fff;
background-color: #fff;
}
应修改 color 属性，如：
css
body {
color: #000;
}
若设置了 text-hide、visibility: hidden 或 display: none 等属性导致文本隐藏，需在开发者工具中找到相关元素并修正这些属性。
内容加载问题：
JavaScript 脚本错误，例如：
javascript
const xhr = new XMLHttpRequest();
xhr.onreadystatechange = function () {
if (this.readyState === 4 && this.status === 200) {
const textContent = this.responseText;
document.getElementById(‘content-area’).innerHTML = textContent;
}
};
xhr.open(‘GET’, ‘data.txt’, true);
xhr.send();
